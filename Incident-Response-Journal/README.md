# Incident Response Journal

## Overview

This project documents multiple simulated cybersecurity incidents and investigative activities completed during hands-on cybersecurity training.

The journal was used to practice structured incident documentation, threat analysis, indicator of compromise (IOC) identification, and application of incident response methodologies commonly used within Security Operations Center (SOC) environments.

---

## Objectives

- Practice incident documentation techniques
- Apply the NIST Incident Response Lifecycle
- Develop investigative and analytical thinking skills
- Identify indicators of compromise (IOCs)
- Improve security monitoring and alert triage capabilities

---

## Incident Scenarios Investigated

### Ransomware Incident
- Investigated a phishing-driven ransomware scenario.
- Analyzed how malicious email attachments can lead to widespread encryption of organizational data.
- Evaluated incident response considerations and potential remediation questions.

### Phishing and Trojan Investigation
- Investigated a password-protected malicious attachment.
- Used SHA256 hash analysis and VirusTotal to validate malicious activity.
- Reviewed IDS alerts and unauthorized executable creation.

### Phishing Alert Ticket Triage
- Investigated a phishing incident affecting an HR employee.
- Analyzed malicious indicators including sender information, file hashes, and attachment details.
- Applied phishing response playbook concepts.

### Customer PII Data Exfiltration Incident
- Investigated unauthorized access to customer information.
- Reviewed web application and web server logs.
- Analyzed a forced browsing attack that resulted in data exposure.
- Evaluated remediation recommendations and long-term security improvements.

## Methodology

Incidents were documented and analyzed using:

### The 5 W's Framework

- Who caused the incident?
- What happened?
- When did it occur?
- Where did it occur?
- Why did it occur?

### NIST Incident Response Lifecycle

1. Preparation
2. Detection and Analysis
3. Containment
4. Eradication
5. Recovery
6. Lessons Learned

## Key Takeaways

- Documentation plays a critical role during investigations.
- Indicators of compromise help identify malicious activity.
- Hash analysis and threat intelligence resources can quickly validate suspicious files.
- Structured incident response methodologies improve consistency and decision-making.
- Effective security operations require both technical analysis and disciplined reporting.
